Kali Linux笔记(三十七)：Nikto

发表于 2019-12-27 | 分类于 Kali

1. NIKTO

  • Perl语言开发的开源web安全扫描器
  • 扫描软件版本
  • 扫描搜索存在安全隐患的文件（压缩包留下的未压缩文件）
  • 扫描服务器配置漏洞（默认配置文件存在的漏洞）
  • WEB Application层面的安全隐患
  • 避免404误判
    • 很多服务器不遵守RFC标准，对于不存在的对象返回200响应码
    • 依据响应文件内容判断，不同扩展名的文件404响应内容不同
    • 去除时间信息后的内容取MD5值哈希摘要（对于404响应内容）
    • -no404（每次扫描前不会尝试扩展名的判断，考虑性能可使用）
改进后的NIKTO：在扫描之前，会从软件数据库中，提取各自扩展名组成随机页面请求，判断是否存在，然后根据不存在的页面去除时间信息后的静态内容取MD5哈希摘要，作为扫描结果404的比对，来进行一定程度的规避404的误判，提高扫描准确性

2. NIKTO

  • 🔺命令：nikto -list -plugins（插件列表）
  • 🔺命令：nikto -update（官方网站对插件的更新，可能需要代理）▲ 2019.12.23 需要通过pull directly from git
    • cirt.net
    • http://cirt.net/nikto/UPDATES
    • Plugins：shellshock（破壳漏洞注意）
  • 🔺命令：nikto -host http://1.1.1.1 -output
  • 🔺命令：nikto -host 192.168.1.1 -ssl -port 443,8443,995
  • 🔺命令：nikto -host host.txt（一行表示一个要扫描的主机）
  • 🔺命令：nmap -p80 192.168.1.0/24 -oG（grep输出）-（输出结果）| nikto -host - ▲ 2019.12.23 语句通过管道连接报错，单独语句可以执行
  • 🔺命令：nikto -host 192.168.1.1 -userproxy http://localhost:22716（本机代理）
  • -vhost（虚拟主机，一个IP多个主机，通过主机头判断，扫描需要域名，不可IP地址）

3. NIKTO–interactive（扫描过程的交互，停止可以再次按命令）

  • Space–report current scan status（报告当前扫描状态）
  • v–verbose mode on/off（显示详细信息）
  • d–debug mode on/off（最详细信息，调试信息，发送的http请求）
  • e–error reporting on/off（错误报告显示）
  • p–progress reporting on/off（进程信息显示）
  • r–redirect display on/off（重定向显示）
  • c–cookie display on/off（服务器所给的cookie显示）
  • a–auth display on/off（身份认证过程显示）
  • q–quit（退出扫描）
  • N–next host（下一个主机）
  • P–Pauser（暂停扫描）
NIKTO不支持表单账号密码身份认证，只支持https身份认证（id+，id:pass:realm），但可以通过设置指定Cookie进行扫描Web Applicattion，需要设置配置文件

4. NIKTO 配置

  • 默认配置文件（/etc/nikto.conf）
  • 需要注意的参数
    • USERAGENT（客户端代理，默认设置Nikto/@VERSION）
    • RFIURL（远程文件包含默认文件，可自己修建自己的URL）
    • CIRT（默认更新的IP地址）
    • PROXYHOST/PROXYPORT/PROXYUSER/PROXYPASS（代理，可以修改代理）
    • STATIC-COOKIE=”cookie1”=”cookie value”;”cookie2”=”cooke val”（设置cookie）
  • -evasion：使用LibWhisker对IDS的躲避技术，可使用以下几种类型（可指定多种）：
    • Random URL encoding(non-UTF8)–随机URL编码（非UTF-8方式 ）
    • Directory self-reference(/./)–自选择路径（/./）
    • Premature URL ending– 过早结束的URL
    • Prepend long random string–优先考虑长随机字符串
    • Fake parameter–参数欺骗
    • TAB as request space–使用TAB作为命令的分隔符
    • Change the case of the URL–使用变化的URL
    • Use Windows directory separator()–使用Windows路径分隔符”"
Cookie获取：
Firefox浏览器，可以通过Firbug option网络抓包，Net，然后Cookie，可通过d-debug mod查看设置的cookie是否生效
User Agent：
Firefox的默认UserAgent：Default User Agent → Default User Agent（每个浏览器可以进行独立设置）
