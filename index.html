003：Kali Linux安装-持久加密USB安装、熟悉BASH命令

发表于 2019-11-20 | 分类于 Kali
  1. 持久加密USB安装

共享文件夹：/medie/sf_D_Drive（物理主机目录）（2.0版本在/mnt/hfgs下）

  • LUKS：Linux Unified Key Setup（基于Linux平台开发）
    • 磁盘分区加密规范
    • 不依赖于操作系统的磁盘级加密
    • Windows-DoxBox（通过前端后端组件进行加密）
    • 后端：dm-crypt
    • 前端：cryptsetup
    • 微软的bitlocker
  • 将镜像刻录到U盘

1.操作系统的删除文件名，只是删除索引，可通过数据还原技术再恢复

(1)文件拷贝：拷贝单个文件到设备

(2)块拷贝：将原磁盘每一个扇区存取的每一块数据完全镜像复制到设备（电子取证）

  • 将镜像刻录到U盘

    🔺命令：dd if=kail-linux-1.1.0-amd64.iso of=/dev/sdb bs=1M

    if：input file

    of：out file bs（复制速度）

    dd(以基于块的模式拷贝到块设备，不同基于文件拷贝)

    🔺命令：dmesg查看系统操作变化

    • 查看插入的u盘所在的位置/sdx
    • 查看U盘原有数据（系统工具-系统管理-Gpatted分区管理器）
    • 卸载分区
    • 删除分区
    • 应用操作
    • 应用dd命令，以块的形式，拷贝到U盘（U盘剩下的空间用于系统加密，若有进行操作，将操作记录在加密部分）

    🔺命令：watch -n 5 killall USR1 dd(汇报工作进展）

1.Window下的efs加密–文件级别加密，用管理员登陆，可以看到文件加密的信息

2.磁盘分区级别的加密，需要密钥，需要证书才可以解出

  1. 持久加密USB安装-2

    • 为U盘创建额外分区
      • 🔺命令：partx分区工具
      • 🔺命令：printf devices列出操作系统挂载的硬盘
      • 🔺命令：select /dev/sdx选择要操作的U盘
      • 🔺命令：mkpart primary起始位置（上一个分区位置）结束位置（分区大小）
      • 🔺命令：quit退出分区工具
    • 使用LUKS加密分区
      • 🔺命令：cryptsetup --verbose --verify-passphrase luksFormat /dev/sdx+分区
      • 输入保护密码
    • 打开加密分区
      • 🔺命令：cryptsetup luksOpen /dev/sdx+分区 usb（可读名称）
      • 🔺命令：ls /dev/mapper/usb（可读名称）
    • 格式化加密分区+卷标
      • 🔺命令：mkfs.ext4（目前最新版本） /dev/mapper/usb（可读名称）
      • 🔺命令：e2label /dev/mapper/usb（可读名称）persistence（官方指定卷标）
    • 挂载加密分区，并创建persistence.conf文件
      • 🔺命令：mkdir -p /mnt/usb
      • 🔺命令：mount /dev/mapper/usb /mnt/usb/
      • 🔺命令：echo "/ union" > /mnt/usb/persistence.conf
      • 🔺命令：umount /dev/mapper/usb
      • 🔺命令：cryptsetup luksClose /dev/mapper/usb

001：Kali Linux渗透测试介绍

发表于 2019-11-20 | 分类于 Kali

  1. 安全问题的根源

    • 分层（固守层面，在一个固定的圈子里）
      • 软件工程师，软件开发
      • 数据库管理师，数据库管理以及数据库管理软件的安装与实现
      • 运维
    • 只追求功能的实现
    • 最大的危险还是人

  2. 安全目标

    • 先于攻击者发现和防止漏洞出现
      • 攻击型安全（攻击手段探测漏洞，主动）
      • 防护型安全（投入巨大，周期很大，安全层度脆碎）

3.渗透测试

  • 尝试挫败安全防御机制，发现系统安全弱点
  • 从攻击者的角度思考，测量安全防护有效性
  • 证明安全问题的存在，而非破坏（深入了解漏洞价值）
  • 道德约束
  • 法律

4.渗透测试标准

  • PETS（http://www.pentest-standard.org）
  • 前期交互交互阶段（渗透测试范围，每次一个子系统的渗透测试，子系统可以为一个应用系统，要进行简单的信息交互）
  • 情报收集阶段（目标系统，邮箱地址，联系人，DNS，ip，公司动态等，对目标系统进行主动的探测，端口探测，主机探测）
  • 威胁建模阶段（轮廓性认识确定一条两条效率最高的途径）
  • 漏洞分析阶段（软件版本是否存在漏洞，软件逆向，形成针对性代码）
  • 渗透攻击阶段（阶段不一定顺利）
  • 后渗透测试阶段（拿到服务器后，进一步的渗透，获取有价值的信息，很重要，危害大）
  • 渗透测试报告

5.渗透测试标准

  • 渗透测试范围
  • 获得授权
  • 渗透测试方法
    • 是否允许社会工程学
    • 是否允许拒绝服务攻击

6.渗透测试误区

  • 扫描器就是一切
    • 忽视业务逻辑重的逻辑（普通员工有获取经历级别的信息的权限）

7.KALI Linux介绍

  • 基于Debian的Linux发行版本
  • 前身是BackTrack，2013年3月发布
  • 用于渗透测试和安全审计
  • 包含600+安全工具
  • FHS标准目录结构（/etc，/bin，/dev，/var等,BT改版的主要原因）
  • 定制内核
  • 支持ARM和手机平台（for ARM树莓派CD卡烧录）
  • 开源免费

8.KALI Linux策略

  • Root策略（大量渗透测试需要root权限，不同于其他Linux平台）
  • 网络服务策略（网络服务默认是关闭的，无自动启动脚本）
  • 更新升级策略（同Debian漏洞升级一样）
