024：Kali Linux主动信息收集-发现

发表于 2019-12-01 | 分类于 Kali

  1. 主动信息收集

    • 直接与目标系统交互通信
    • 无法避免留下访问的痕迹（可能被封杀，强的流量分析，强的代码审计）
    • 使用受控的第三方电脑进行探测
      • 使用代理或已被控制的主机
      • 做好被封杀的准本
      • 使用噪声迷惑目标，淹没真实的探测流量（伪造IP发送大量垃圾信息）
    • 扫描（IP层 → 端口层 → 服务级别的扫描）
      • 发送不同的探测，根据结果判断目标状态

  2. 发现（IP层 → 端口层 → 服务级别的扫描）

    • 识别活着的主机
      • 潜在的被攻击目标（存在的漏洞来进行利用）
    • 输出一个IP地址列表（在线IP地址）
    • 2、3、4层发现

  3. 发现-二层发现

    • 优点：扫描速度快、可靠
    • 缺点：不可路由（只可发现本网段的主机发现）
    • Arp协议
      • 抓包
OSI模型
Layer1 Physical
Layer2 Data Link       ARP
Layer3 Network         IPv4,IPv6,ICMP,and IPSec
Layer4 Transport       TCP and UDP
Layer5 Session         NetBIOS,PPTP,RPC,and SOCKS
Layer6 Presentation    ASCII,JPEG,PDF,PNG,and DOCX
Layer7 Application     HTTP,FTP,and Telnet
  • arping
  • arping IP地址 -c 1（发送一个ARP包）
  • arping IP地址 -d（发现重复IP地址拥有一个MAC地址，可以发现arp地址欺骗）
  • arping -c 1 Ip地址 | grep “bytes from” | cut -d “ “ -f 5 | cut -d “(“ -f 2 | cut -d “)” -f 1
  • 脚本 ▲2019.12.1新版ifconfig命令显示不同
    • arping arping1.sh eth0 >addr.txt（检查网内”存活”的计算机）
  • 脚本
    • arping arping2.sh eth0（重新检查”存活”的主机）

  1. 发现-二层发现

    • nmap IP地址-254（IP地址/24） -sn
    • namp -iL（调用地址列表的文本文件） iplist.txt -sn
强大的NMAP软件，需要非常熟练
nmap的-sn参数，除了会ARP发现之外，还会进行DNS的反向域名解析
